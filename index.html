Use our templates and examples to generate your rules files with ChatGPT:

"Using the base structure in cursor-rules-structure/ and following the examples in examples/, can you help me create rules files for my project in the .cursor/rules/ directory? Please maintain the same structure but adapt it to my project's specific needs."

Using Cursor Effectively

1. Project Setup

  • Create a new repository
  • Initialize a basic project (e.g., Next.js app)
  • Add .cursor/rules directory to the root
  • Create separate rule files for different aspects (e.g., design, development)

Pro Tip: Use the globs option in your rule files to make them specific to certain file patterns: 

---
description: Style guidelines for React components
globs: 
  - src/components/**/*.tsx
  - src/components/**/*.jsx
---

## Component Style Rules
  - Use functional components
  - Follow BEM naming convention
  - Implement responsive design

This rule file will only be applied when working with React component files in the specified directories.

2. Configure Documentation

  • Check Cursor's default documentation list
  • Add custom documentation if needed
  • Reload Cursor for proper indexing

3. Optimal Usage

  • Use Composer + Agent for code generation

Migrating from .cursorrules

Follow these steps to migrate your existing .cursorrules file to the new structured format:

1. Copy the Base Structure

Start by copying our base structure from the cursor-rules-structure directory to your project:

my-project/
├── .cursor/
│   └── rules/
│       ├── overview.mdc
│       ├── tech-stack.mdc
│       ├── design-rules.mdc
│       ├── areas.mdc
│       └── development-guidelines.mdc

2. Use Cursor to Update Rules

Ask Cursor to help migrate your existing rules:

"Based on my existing @.cursorrules file, can you help me update each file in .cursor/rules/ with my project's specific details? Please maintain the same structure but adapt the content to match my project's needs."

3. Review and Enhance

  • Verify that all important information was transferred
  • Add any missing project-specific guidelines
  • Update file globs to match your project structure
  • Remove any irrelevant sections

Pro Tip: Gradual Migration

You can keep your old .cursorrules file while transitioning to the new structure. This allows you to migrate gradually and ensure everything works as expected before removing the old file.

Troubleshooting & Best Practices

Refining Generated Code

Example Scenario: Improving a Login Page

  1. Take a screenshot of the current implementation
  2. Find a reference design you prefer
  3. In Composer, explain:
    "I'd like to modify the login page (@components/auth/LoginForm.tsx) to match this design style [reference screenshot]. Specifically, I prefer:
    - The centered card layout
    - Minimal color scheme
    - Larger input fields"

Effective Communication with Cursor

  • Use @filename to reference specific files
    "Please update @components/Header.tsx to include the new navigation items"
  • Keep context focused and relevant
    ❌ "Here's my entire component library with 50 files, please update the button..."
    ✅ "Update the button styles in @components/Button.tsx to use the primary color"
  • Reference official documentation
    "Following the Tailwind CSS docs in Cursor, let's implement a responsive navbar..."
