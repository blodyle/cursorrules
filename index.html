Create Your .cursorrules File

A step-by-step guide to structuring your project with Cursor AI

1. Find Your Idea

Start with a clear vision of what you want to build. Consider:

  • The problem you're solving
  • Your target users
  • Core functionality needed
  • Technical requirements

2. Create a Detailed Plan

Use ChatGPT to help create a comprehensive project plan. Ask for:

"Can you help me create a detailed plan for [your project]? Please include:
- Core features and functionality
- Key areas of the application
- Target audience analysis
- Usability guidelines and best practices
- Technical requirements and constraints"

3. Generate .cursorrules File

Use our template and example to generate your .cursorrules file with ChatGPT:

"Using this template [base-structure] and following this example [.cursorrules-example], can you help me create a .cursorrules file for [your project]? Please maintain the same structure but adapt it to my project's specific needs. Write it in a canvas."

Important Note

This guide is meant to be a starting point, not a rigid framework. Here's what you should keep in mind:

Every project is unique, with its own requirements, challenges, and quirks. Adapt these guidelines to fit your specific needs.

A technical background is invaluable for properly structuring your application, especially when it comes to separating responsibilities (frontend, backend, database access) and maintaining clean architecture.

While Cursor AI significantly speeds up development, understanding good coding practices helps with debugging and long-term maintainability.

Most importantly, have fun! Use this as a learning opportunity to experiment with new approaches and find what works best for you and your team.

Using Cursor Effectively

1. Project Setup

  • Create a new repository
  • Initialize a basic project (e.g., Next.js app)
  • Add .cursorrules file to the root
  • Create separate .cursorrules for different tech stacks (e.g., frontend/backend)

2. Configure Documentation

  • Check Cursor's default documentation list
  • Add custom documentation if needed
  • Reload Cursor for proper indexing
View Default Documentation List →

3. Optimal Usage

  • Use Composer + Agent for code generation
  • Prefer Claude-3.5-sonnet model for better results
  • Use Chat for ideation and planning
  • Keep .cursorrules updated as project evolves

Pro Tip

  • Always reload Cursor after adding new documentation or making significant changes to your .cursorrules file. This ensures better context awareness and more accurate suggestions.

Troubleshooting & Best Practices

Refining Generated Code

Example Scenario: Improving a Login Page

  1. Take a screenshot of the current implementation
  2. Find a reference design you prefer
  3. In Composer, explain:
    "I'd like to modify the login page (@components/auth/LoginForm.tsx) to match this design style [reference screenshot]. Specifically, I prefer:
    - The centered card layout
    - Minimal color scheme
    - Larger input fields"

Effective Communication with Cursor

  • Use @filename to reference specific files
    "Please update @components/Header.tsx to include the new navigation items"
  • Keep context focused and relevant
    ❌ "Here's my entire component library with 50 files, please update the button..."
    ✅ "Update the button styles in @components/Button.tsx to use the primary color"
  • Reference official documentation
    "Following the Tailwind CSS docs in Cursor, let's implement a responsive navbar..."

Pro Tips

  • Break down large changes into smaller, focused requests
  • Use screenshots and examples for visual changes
  • Keep the .cursorrules file updated as your project evolves
  • Regularly check Cursor's documentation list for updates
